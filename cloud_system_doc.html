基于 OpenGL 4.x · 单 Pass Raymarching · 覆盖 Earth 行星

1. 系统概览

云系统完全在 GLSL fragment shader 内实现，随大气层 drawAtmosphere() 调用一同执行。主要设计目标：从轨道视角（500–50000 km）看到接近真实的地球云层， 同时在低轨（<200 km）仍保持合理的云体结构。

2. Coverage Map（覆盖率贴图）

Coverage 是一张定义在球面上的 2D 标量场，决定云在哪里存在。 它与 3D 形状噪声完全解耦：形状噪声决定云的轮廓细节，coverage 决定每个经纬度的宏观密度权重。

2.1 计算流程

→

→

线性 Ramp [covLo, covHi]
→
coverage ∈ [0,1]

2.2 Curl Noise Domain Warp（核心）

用标量势场 φ 的旋度（curl）来扭曲采样坐标。旋度场是散度为零的向量场， 其流线自然形成闭合环路——即涡旋结构，无需手动放置任何气旋。

数学原理： 对球面切平面上的标量势 φ，旋度 = ∇φ 旋转 90°：

curlU = -∂φ/∂v   (v 方向梯度取反)
curlV = +∂φ/∂u   (u 方向梯度)

→ 流线是 φ 的等值线（绕极值点的闭合轨道 = 涡旋）

实现（有限差分，4 次采样）：

// 低频势场（~4000km 涡旋）
vec3 seedLo = vec3(t*0.003, 0.0, t*0.002);
float pLo_pu = fbm(coverageSph * 1.5 + wAxis * EPS + seedLo, 2);
float pLo_mu = fbm(coverageSph * 1.5 - wAxis * EPS + seedLo, 2);
float pLo_pv = fbm(coverageSph * 1.5 + vAxis * EPS + seedLo, 2);
float pLo_mv = fbm(coverageSph * 1.5 - vAxis * EPS + seedLo, 2);
float curlU_lo = -(pLo_pv - pLo_mv);   // 旋转梯度 90°
float curlV_lo =  (pLo_pu - pLo_mu);

// 高频势场（~1000km 中尺度涡）
vec3 seedHi = vec3(91.2 + t*0.005, 5.1, 22.0);
float pHi_pu = fbm(coverageSph * 6.0 + wAxis * EPS + seedHi, 1);
// ... 同上 4 次采样 ...
float curlU_hi = -(pHi_pv - pHi_mv);
float curlV_hi =  (pHi_pu - pHi_mu);

// 合并并位移采样坐标（EPS=0.06；低频振幅≈0.9，高频≈0.72）
float warpU = curlU_lo * 2.5 + curlU_hi * 1.0;
float warpV = curlV_lo * 2.5 + curlV_hi * 1.0;
vec3 warpedSph = normalize(coverageSph + wAxis * warpU + vAxis * warpV);
关键参数： EPS = 0.06（有限差分步长）； 低频 scale=1.5 对应涡旋直径 ~4000km，高频 scale=6.0 对应 ~1000km。 scale 越高，涡旋越小越多。raw curl 幅度 ≈ ±(2·EPS·gradient)， gradient ≈ 2.0（2 oct）或 6.0（1 oct，scale=6.0）。

2.3 FBM 覆盖率采样

// 大尺度云团（~8000km 以上）
float cov_large  = fbm(warpedSph * 0.8  + vec3(t*0.004),           5 octaves);

// 中尺度扰动（~1000km）
float cov_medium = fbm(warpedSph * 6.0  + vec3(t*0.011, -t*0.008, t*0.013), 3 octaves);

float coverage = cov_large * 0.65 + cov_medium * 0.35;

// 独立纬度 warp（避免锋面成直线带状）
float latNoise  = fbm(coverageSph * 4.0 + vec3(127.3, 88.1, 9.6+t*0.006), 3) - 0.5;
float warpedLat = sph.z + latNoise * 0.40;   // ±40% 纬度抖动

2.4 纬度分布模型（latCloud）

5 个 Gaussian 峰模拟真实地球云带分布，1 个 Gaussian 减法模拟副热带高压晴空区：

float lat = warpedLat;
float itczB = exp(-(lat/0.34)²)                     // 赤道辐合带 (ITCZ)
float ntmpB = exp(-((lat-0.766)/0.42)²)             // 北半球温带锋面 (~44°N)
float stmpB = exp(-((lat+0.766)/0.42)²)             // 南半球温带锋面 (~44°S)
float npolB = exp(-((lat-0.940)/0.34)²)             // 北极涡旋 (~70°N)
float spolB = exp(-((lat+0.940)/0.34)²)             // 南极涡旋 (~70°S)
float subDry= exp(-((abs(lat)-0.50)/0.28)²)         // 副热带干燥带 (~30°N/S)

float latCloud = clamp(
    0.28 + 0.45*itczB + 0.38*ntmpB + 0.38*stmpB
          + 0.20*npolB + 0.20*spolB - 0.12*subDry,
    0.28, 1.0);     // 下限 0.28：任意纬度不低于此值
参数值含义
subDry 系数0.12副热带清除强度（越大越暗）
subDry 宽度0.28副热带宽度（越大越平滑，无硬边）
clamp 下限0.28最暗纬度的底色（防止全黑条带）

2.5 Coverage → Density 映射

// latCloud 乘法压缩后做线性 ramp
coverage = clamp(coverage * latCloud, 0.0, 1.0);

// 线性 ramp（不用 smoothstep — S 曲线会把 coverage 推向 0/1，恢复二值化）
float covRange = max(uTuneCovHi - uTuneCovLo, 0.001);
coverage = clamp((coverage - uTuneCovLo) / covRange, 0.0, 1.0);
为什么用线性而不是 smoothstep： smoothstep 的 S 型导数在两端趋近 0， 会将大量中间值推向 0 或 1，导致 density 重新变成"全有或全无"。 线性 ramp 保留 coverage 的连续梯度，使薄云区域真正有 20%–60% 的密度。

3. 形状噪声与密度计算

3.1 坐标体系

// 纬度相关的风场旋转（科里奥利效应近似）
float latWarp = sph.z³ * 0.3;
float s1 = sin(latWarp)*cos + cos(latWarp)*sin;   // 纬度混合旋转角
vec3 windSph1 = rotate(sph, s1, c1);              // 形状噪声的采样坐标

// 高度方向编码：球面径向扩展
vec3 tc = windSph1 * (80.0 + h*0.45) + vec3(t*0.6, t*0.4, t*0.15);

3.2 高度轮廓

// 底部线性上升（云底）
float altBase = smoothstep(gCloudMinAlt, gCloudMinAlt + 1.5, h);

// 顶部指数衰减（云顶），每个云簇顶高由 topNoise 控制
float topNoise = fbm(windSph1 * 40.0, 2);           // ~160km 尺度
float topExp   = mix(0.85, 2.6, topNoise);           // 0.85=高积雨云, 2.6=矮积云
float altDome  = clamp(1.0 - pow(altNorm, topExp), 0.0, 1.0);
float altShape = altBase * altDome * 1.5;

3.3 3D 形状 FBM（Layer 1）

// 基础形状（6 oct，带域变形 warp）
float warp = fbm(tc * 0.3, 3);
float n = fbm(tc + vec3(warp*0.4, warp*0.4, warp*0.2) + vec3(t*0.02), 6);

// 侵蚀（Erosion）：高频减法，使云边缘破碎
float ero = fbm(tc * 2.3 + vec3(-t*0.04, t*0.03, 0.0), 4);
n -= ero * uTuneErosion;

// LOD 细节（仅近距离）
if (viewDist < 250km)   n += Worley cells * 0.16;    // ~160km 花椰菜结构
if (viewDist < 80km)    n += fbm(p*0.3, 3) * 0.13;   // 3km 凸起
if (viewDist < 10km)    n += fbm(p*2.5, 3) * 0.10;   // 沸腾纹理
if (viewDist < 1.5km)   n += fbm(p*18.0, 2) * 0.05;  // 微细节

3.4 密度计算（关键路径）

// 固定阈值 mask（不随 coverage 变化）+ coverage 作为密度乘子
float maskLo = min(uTuneThreshLo, uTuneThreshHi);   // 防止 edge0 >= edge1
float maskHi = max(uTuneThreshLo, uTuneThreshHi);
float mask = smoothstep(maskLo, maskHi, n) * altShape;
density = max(density, mask * uTuneDensity * coverage);
设计原理： coverage 为 0 时 density 恒为 0（数学上完全消除散点）； coverage = 0.4 时同一朵云的密度降为 40%（薄云效果）；coverage = 1 时满密度云。 这与旧方案（coverage 控制 threshold）的根本区别是： 旧方案在低 coverage 区域约 15% 的 FBM 体素仍能通过阈值，产生散点； 新方案不论 n 多大，coverage=0 的地方永远没有云。

4. 气旋引力子（Cyclone Attractor）

4.1 实现思路

气旋不是独立渲染对象，而是通过两个阶段影响云的 coverage 场：

  • 阶段 1（域扭转）： 在 FBM 形状噪声采样前，用 Rodrigues 旋转扭曲 windSph1， 使噪声场在气旋中心附近产生螺旋臂。
  • 阶段 2（coverage 增强）： 乘法增强气旋区域的 coverage（不引入白圆环）， 并用带 FBM 噪声的边界挖出台风眼。

4.2 代码实现

// 气旋中心随时间绕 z=0.35 纬度圈公转
float cycAngle = t * -0.1;
vec3 cycCenter = normalize(vec3(0.6*cos(cycAngle), 0.6*sin(cycAngle), 0.35));
float cycDist  = distance(sph, cycCenter);      // 世界空间距离（不是风场坐标）
float cycRadius = 0.22;
float cycInfluence = smoothstep(cycRadius, 0.0, cycDist);   // 线性衰减

// 域扭转：~103° 最大旋转（旧 459° 使噪声完全模糊成圆盘）
if (cycInfluence > 0.001) {
    float twist = cycInfluence * 1.8;   // 0 ~ 103°
    float ct = cos(twist), st = sin(twist);
    // cycCenter 转入风场坐标系（与 windSph1 一致）
    vec3 cycCenterW = normalize(rotate(cycCenter, c1, s1));
    vec3 warped = Rodrigues(windSph1, cycCenterW, twist);
    windSph1 = mix(windSph1, warped, cycInfluence);
}

// Coverage 增强（阶段 2，在 remap 之后）
if (cycInfluence > 0.001) {
    coverage = mix(coverage, clamp(coverage*1.9, 0,1), cycInfluence*0.7);
    // 台风眼：FBM 毛边（不是完美圆）
    float eyeNoise  = fbm(sph * 38.0 + vec3(t*0.02), 2) * 0.009;
    float eyeRadius = 0.017 + eyeNoise;
    coverage *= smoothstep(eyeRadius * 0.35, eyeRadius, cycDist);
}
坐标系 Bug（已修复）： 旧代码 cycDist = distance(windSph1, cycCenter) 把风场坐标（旋转后）与世界坐标（未旋转）做比较，气旋引力永远定位错误。 修复后：距离在世界空间测量，Rodrigues 旋转轴才转入风场坐标系。

5. 光照模型

5.1 Beer-Lambert 自阴影

// 向太阳方向 8 步光照 march
float optDepthC = ...;   // 当前点到云顶的光学深度
float cLightC   = ...;   // 太阳光 march 积分

float tauCloud = gCloudExtinction.x * (optDepthC + cLightC);

5.2 多散射近似（Jimenez 4-term）

float ms0 = exp(-tauCloud);
float ms1 = exp(-tauCloud * 0.5) * 0.5;
float ms2 = exp(-tauCloud * 0.25) * 0.25;
float ms3 = exp(-tauCloud * 0.125) * 0.125;
float multiScatter = (ms0 + ms1 + ms2 + ms3) / 1.875;

5.3 Powder Effect（银边效应）

float powder    = 1.0 - exp(-tauCloud * 2.0);
float silverMask = max(0.0, cosTheta);   // 仅向阳面生效（物理正确）
float cloudAtt  = multiScatter * mix(1.0, powder, 0.55 * silverMask);
silverMask = max(0, cosTheta) 确保 powder 增亮只对向太阳方向的射线生效。 旧代码对全部射线应用 powder，削弱了背光侧的阴影对比度。

5.4 相函数

// Henyey-Greenstein 双叶混合
float phase1 = HG(cosTheta, g=0.82);    // 强前向散射（主瓣）
float phase2 = HG(cosTheta, g=-0.3);    // 弱后向散射（银边）
float phase  = mix(phase2, phase1, w);  // w 由视角决定

6.1 参数说明

6.2 调参策略

  • 云太少 / 晴空太多： 降低 CovLo，扩大 [CovLo, CovHi] 区间
  • 云太均匀、缺乏对比： 提高 CovLo，缩窄区间
  • 散点（晴空里有孤立小云）： coverage 乘子机制从根本上消除此问题；若仍出现，检查 mask min/max 是否正确
  • 云太厚全白： 降低 Density 或 Extinction
  • 云边缘太硬： 增大 ThreshHi - ThreshLo 的区间，或增大 Erosion
  • 纬度条带明显： 在代码中减小 subDry 系数或提高 latCloud clamp 下限

7. Debug 工具

7.1 F3 键切换模式

按 F3 次数模式输出用途
1 次Coverage 灰度图Raw coverage（smoothstep 前）确认云分布形态：有无螺旋、条带是否过重
2 次Curl 向量场彩色图R=curlU, G=curlV, B=0.5诊断 curl noise 是否产生涡旋
3 次正常渲染—恢复

7.2 Curl 彩色图判读

  • 颜色绕某点旋转（红→橙→绿→青循环） → curl 正确，涡旋产生中
  • 大片均匀色块，颜色单调 → 涡旋太大（scale 太低）或振幅太小
  • 椒盐噪点（相邻像素剧烈跳变） → EPS 太小，有限差分精度不足

7.3 Coverage 灰度图判读

  • 水平条带明显 → latCloud 参数过强，降低 subDry 系数或提高 clamp 下限
  • 整体很暗（值集中在 0–0.15） → latCloud 乘法压缩过重；降低 covLo 或修改 latCloud 乘法为混合模式
  • 绿色异常亮斑 → 气旋 coverage boost 异常或坐标系 bug
  • 无结构（均匀灰） → curl warp 振幅不足，检查 EPS 和 scale 设置
